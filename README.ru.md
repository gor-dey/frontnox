# FrontNox

[![English](https://img.shields.io/badge/lang-English-blue)](README.md)

Легковесный набор инструментов для фронтенд-разработчиков. Чистый PowerShell, без зависимостей.

**Два инструмента, один установщик:**

| Инструмент | Описание |
|------------|----------|
| `proj` | Менеджер проектов — сохраняй, переключайся и запускай проекты |
| `corsproxy` | Локальный CORS-прокси — обходи CORS-ограничения при разработке |

> 🌐 Двуязычный интерфейс: English & Русский

---

## Установка

### через winget *(рекомендуется)*

```
winget install gor-dey.FrontNox
```

> Устанавливает все инструменты без запросов, на английском. После установки перезапустите терминал.

### Вручную

**Двойной клик** по `Install.cmd` — или вручную:

```powershell
pwsh .\scripts\Install.ps1
# или
powershell .\scripts\Install.ps1
```

Флаги для тихой установки:

| Флаг | Описание |
|------|----------|
| `-All` | Установить все инструменты без запросов |
| `-Lang ru` | Пропустить выбор языка (укажи `en` или `ru`) |

```powershell
# Установить всё без запросов на русском
pwsh .\scripts\Install.ps1 -All -Lang ru
```

Установщик:

1. Предложит выбрать язык (`en` / `ru` — переключение через <kbd>Tab</kbd>) — пропускается при `-Lang`
2. Позволит выбрать, какие инструменты установить — пропускается при `-All`
3. Скопирует скрипты в `~/.config/FrontNox/bin/`
4. Обновит профиль PowerShell

После установки **перезапустите терминал**.

### Удаление

**Двойной клик** по `Uninstall.cmd` — или:

```powershell
pwsh .\scripts\Uninstall.ps1
```

## `proj` — Менеджер проектов

Сохраняй директории как именованные проекты, переходи к ним или запускай `npm run dev` одной командой.

### Команды

#### `proj list`

Показать все зарегистрированные проекты.

```
📋 Зарегистрированные проекты:
  myapp -> C:\Users\me\projects\myapp
  landing -> C:\Users\me\projects\landing
```

#### `proj add <имя>`

Сохранить текущую директорию как проект.

```powershell
cd C:\Users\me\projects\myapp
proj add myapp
# ✅ Проект 'myapp' сохранён.
```

#### `proj go <имя>`

Перейти в директорию проекта.

```powershell
proj go myapp
# переход в C:\Users\me\projects\myapp
```

#### `proj run <имя>`

Перейти в проект и запустить его:

- Автоопределение пакетного менеджера (`npm`, `yarn`, `pnpm`, `bun`) по lock-файлу
- Выполнит install, если `node_modules/` отсутствует
- Запустит `dev` (или `start` как запасной вариант)

```powershell
proj run myapp
# 🚀 Запуск: C:\Users\me\projects\myapp
# ▶️ Запуск 'npm run dev'...
```

#### `proj remove <имя>`

Удалить проект из списка (файлы не удаляются).

```powershell
proj remove myapp
# 🗑️ Проект 'myapp' удалён.
```

#### `proj rename <старое-имя> <новое-имя>`

Переименовать проект.

```powershell
proj rename myapp frontend
# ✅ Проект 'myapp' переименован в 'frontend'.
```

#### `proj -Version` / `corsproxy -Version`

Показать установленную версию.

> **Подсказка:** Все имена проектов поддерживают автодополнение через <kbd>Tab</kbd>.

## `corsproxy` — CORS-прокси

Локальный HTTP-прокси, добавляющий CORS-заголовки к любому ответу. Направьте запросы фронтенда на `http://localhost:8080/<целевой-url>` вместо прямого обращения к API.

### Использование

```powershell
corsproxy                    # Порт 8080, логирование только ошибок
corsproxy -Port 3001         # Свой порт
corsproxy -ShowAll           # Логировать все запросы
corsproxy -NoLog             # Без файлового логирования
```

### Как это работает

Фронтенд отправляет запрос через прокси:

```
GET http://localhost:8080/https://api.example.com/data
```

Прокси перенаправляет запрос на `https://api.example.com/data`, добавляет CORS-заголовки к ответу и возвращает его вашему приложению. Preflight-запросы (`OPTIONS`) обрабатываются автоматически.

### Логирование

По умолчанию прокси логирует ошибочные ответы (статус 400+) как JSON-файлы в `%TEMP%\NoxProxyLogs\`. Каждый файл содержит:

- Время и длительность запроса
- Метод, URL, заголовки и тело запроса
- Статус, заголовки и тело ответа

Кликабельные ссылки на файлы отображаются в терминале для быстрого доступа.

| Флаг | Поведение |
|------|-----------|
| *(по умолчанию)* | Только ошибки (400+) |
| `-ShowAll` | Все запросы |
| `-NoLog` | Без файлового логирования |

## Структура проекта

```
frontnox/
├── Install.cmd          # Лаунчер (двойной клик для установки)
├── Uninstall.cmd        # Лаунчер (двойной клик для удаления)
├── scripts/
│   ├── Install.ps1      # Скрипт установки
│   └── Uninstall.ps1    # Скрипт удаления
└── src/
    ├── constants.ps1    # Общие пути и конфигурация
    ├── proj.ps1         # Менеджер проектов
    ├── corsproxy.ps1    # CORS-прокси
    └── i18n/
        ├── en.json      # Английские строки
        └── ru.json      # Русские строки
```

После установки скрипты хранятся в:

```
~/.config/FrontNox/
├── bin/                 # Скрипты и i18n
├── proj.json            # Сохранённые проекты
└── lang.conf            # Настройка языка
```
